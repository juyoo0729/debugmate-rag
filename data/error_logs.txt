# 오류 1: ModuleNotFoundError: No module named 'datasets'

## 상황

Jupyter Notebook에서 다음 코드를 실행했을 때 오류가 발생했다.

```python
from datasets import Dataset
```

오류 메시지:

```text
ModuleNotFoundError: No module named 'datasets'
```

## 원인

현재 Jupyter 커널이 사용하는 Python 환경에 `datasets` 패키지가 설치되어 있지 않다.

또는 PowerShell에서 패키지를 설치한 Python 환경과 Jupyter Notebook이 사용하는 Python 커널이 다를 수 있다.

## 해결 방법

먼저 현재 Jupyter 커널의 Python 경로를 확인한다.

```python
import sys
print(sys.executable)
```

그다음 현재 커널이 사용하는 Python 환경에 `datasets`를 설치한다.

```powershell
python -m pip install datasets
```

또는 프로젝트의 `requirements.txt`를 기준으로 설치한다.

```powershell
python -m pip install -r requirements.txt
```

## 주의사항

VS Code 터미널에서 설치한 패키지가 Jupyter Notebook 커널에 바로 적용되지 않을 수 있다.

설치 후에는 Jupyter 커널을 재시작해야 한다.

## 관련 태그

package_error, datasets, jupyter, python_env

# 오류 2: ModuleNotFoundError: No module named 'ragas'

## 상황

RAGAS 평가 코드를 실행할 때 다음 import에서 오류가 발생했다.

```python
from ragas import evaluate
```

오류 메시지:

```text
ModuleNotFoundError: No module named 'ragas'
```

## 원인

현재 Python 가상환경에 `ragas` 패키지가 설치되어 있지 않다.

또는 Jupyter Notebook이 프로젝트 가상환경이 아닌 전역 Python을 사용하고 있을 수 있다.

## 해결 방법

현재 가상환경이 활성화되어 있는지 확인한다.

```powershell
python --version
python -c "import sys; print(sys.executable)"
```

그다음 `ragas`를 설치한다.

```powershell
python -m pip install ragas==0.1.21
```

프로젝트 전체 의존성을 설치하려면 다음 명령어를 사용한다.

```powershell
python -m pip install -r requirements.txt
```

## 주의사항

RAGAS는 버전별 API 차이가 있을 수 있다.

강의 코드나 기존 예제가 `ragas==0.1.21` 기준이라면 최신 버전보다 고정 버전을 사용하는 것이 안정적이다.

## 관련 태그

package_error, ragas, rag_evaluation, python_env

# 오류 3: ModuleNotFoundError: No module named 'langchain_core'

## 상황

LangChain 관련 코드를 실행할 때 다음 import에서 오류가 발생했다.

```python
from langchain_core.documents import Document
```

오류 메시지:

```text
ModuleNotFoundError: No module named 'langchain_core'
```

## 원인

`langchain-core` 패키지가 설치되어 있지 않거나, 설치된 LangChain 관련 패키지 버전 조합이 맞지 않는다.

또는 현재 Jupyter 커널이 올바른 가상환경을 사용하지 않고 있을 수 있다.

## 해결 방법

`requirements.txt`에 명시된 버전 조합으로 설치한다.

```powershell
python -m pip install -r requirements.txt
```

개별 설치가 필요하면 다음 명령어를 사용한다.

```powershell
python -m pip install langchain-core==0.1.52
```

## 주의사항

LangChain은 패키지가 여러 개로 분리되어 있다.

주요 패키지:

```text
langchain
langchain-core
langchain-community
langchain-openai
langchain-text-splitters
langchain-chroma
```

하나만 설치되어 있다고 전체 기능이 동작하는 것은 아니다.

## 관련 태그

package_error, langchain, langchain_core, python_env

# 오류 4: ModuleNotFoundError: No module named 'langchain_community.chat_models.vertexai'

## 상황

RAGAS 또는 LangChain 관련 import 과정에서 다음 오류가 발생했다.

```text
ModuleNotFoundError: No module named 'langchain_community.chat_models.vertexai'
```

## 원인

`ragas`, `langchain-community`, `langchain-core` 사이의 버전 조합이 맞지 않을 수 있다.

특정 RAGAS 버전이 기대하는 LangChain community 모듈 구조와 현재 설치된 버전의 구조가 다를 수 있다.

## 해결 방법

기존 패키지를 최신으로 무작정 올리기보다, 안정적으로 동작했던 버전 조합을 사용한다.

권장 버전 예시:

```text
ragas==0.1.21
langchain==0.1.20
langchain-core==0.1.52
langchain-community==0.0.38
langchain-openai==0.1.7
```

설치 명령어:

```powershell
python -m pip install -r requirements.txt
```

## 주의사항

LangChain 계열은 버전 호환성이 중요하다.

오류가 발생하면 무작정 `pip install -U`로 전체 업그레이드하기보다, 프로젝트에서 검증한 버전으로 고정하는 것이 좋다.

## 관련 태그

version_error, ragas, langchain, langchain_community

# 오류 5: OpenAIError: The api_key client option must be set

## 상황

OpenAI Embedding 또는 ChatOpenAI를 실행할 때 오류가 발생했다.

예시 코드:

```python
embeddings = OpenAIEmbeddings()
llm = ChatOpenAI(model="gpt-4o-mini")
```

오류 메시지:

```text
OpenAIError: The api_key client option must be set either by passing api_key to the client or by setting the OPENAI_API_KEY environment variable
```

## 원인

현재 Python 실행 환경에 `OPENAI_API_KEY` 환경변수가 설정되어 있지 않다.

또는 `.env` 파일이 있지만 Notebook이나 Streamlit 앱에서 제대로 로드하지 못했다.

## 해결 방법

프로젝트 루트에 `.env` 파일을 만든다.

```text
OPENAI_API_KEY=본인_API_KEY
```

Python 코드에서 `.env`를 로드한다.

```python
from dotenv import load_dotenv
import os

load_dotenv(".env")

print(os.environ.get("OPENAI_API_KEY") is not None)
```

Notebook이 `notebooks/` 폴더 안에서 실행된다면 다음처럼 상위 폴더의 `.env`를 로드한다.

```python
load_dotenv("../.env")
```

## 주의사항

API 키를 노트북이나 GitHub에 직접 적으면 안 된다.

GitHub에 올리기 전 다음 명령어로 키 문자열이 남아 있는지 확인한다.

```powershell
Select-String -Path .\notebooks\*.ipynb -Pattern "sk-"
```

## 관련 태그

api_key_error, openai, dotenv, security

# 오류 6: SyntaxError: invalid syntax when running python -m venv in notebook

## 상황

Jupyter Notebook 셀에서 다음 명령어를 실행했을 때 오류가 발생했다.

```python
python -m venv ragas-env
```

오류 메시지:

```text
SyntaxError: invalid syntax
```

## 원인

`python -m venv`는 Python 코드가 아니라 터미널 명령어이다.

Jupyter Notebook 코드 셀에 그대로 입력하면 Python 문법으로 해석되어 오류가 난다.

## 해결 방법

PowerShell 또는 터미널에서 실행한다.

```powershell
python -m venv .venv
```

Jupyter Notebook 안에서 실행해야 한다면 앞에 `!`를 붙인다.

```python
!python -m venv .venv
```

하지만 가상환경 생성은 Notebook보다 PowerShell에서 하는 것이 더 안전하다.

## 주의사항

Notebook 셀과 PowerShell 명령어를 구분해야 한다.

Python 코드:

```python
import pandas as pd
```

PowerShell 명령어:

```powershell
python -m pip install pandas
```

## 관련 태그

syntax_error, jupyter, powershell, venv

# 오류 7: PowerShell에서 bash 명령어 인식 실패

## 상황

PowerShell에서 다음 명령어를 실행했을 때 오류가 발생했다.

```powershell
bash setup.sh
```

오류 메시지:

```text
The term 'bash' is not recognized as a name of a cmdlet, function, script file, or executable program.
```

## 원인

Windows PowerShell에는 기본적으로 `bash` 명령어가 없을 수 있다.

Git Bash, WSL, 또는 Linux 환경이 아니라면 `.sh` 스크립트를 직접 실행하기 어렵다.

## 해결 방법

Windows PowerShell용 명령어를 따로 작성해서 실행한다.

예를 들어 가상환경 생성과 패키지 설치는 다음처럼 실행한다.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

또는 Git Bash를 설치한 뒤 Git Bash 터미널에서 실행한다.

## 주의사항

Windows에서는 `.sh`보다 PowerShell 명령어를 사용하는 것이 초보자에게 더 안정적이다.

## 관련 태그

powershell_error, bash, windows, setup

# 오류 8: .venv Activate.ps1 파일을 찾을 수 없음

## 상황

가상환경을 활성화하려고 할 때 오류가 발생했다.

```powershell
.\.venv\Scripts\Activate.ps1
```

오류 메시지:

```text
The term '.\.venv\Scripts\Activate.ps1' is not recognized as a name of a cmdlet, function, script file, or executable program.
```

## 원인

`.venv` 폴더가 없거나, 가상환경 생성이 실패해서 `Scripts\Activate.ps1` 파일이 존재하지 않는다.

또는 현재 PowerShell 위치가 프로젝트 폴더가 아닐 수 있다.

## 해결 방법

먼저 현재 위치를 확인한다.

```powershell
pwd
```

프로젝트 폴더로 이동한다.

```powershell
cd C:\AI_study\debugmate-rag
```

가상환경을 생성한다.

```powershell
python -m venv .venv
```

생성 후 활성화 파일이 있는지 확인한다.

```powershell
Test-Path C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1
```

결과가 `True`이면 활성화한다.

```powershell
(Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1)
```

## 주의사항

VS Code에서 `.venv` 폴더를 직접 만들면 안 된다.

반드시 `python -m venv .venv` 명령어로 만들어야 한다.

## 관련 태그

venv_error, powershell, activate, python_env

# 오류 9: Python 3.12에서 chroma-hnswlib 빌드 실패

## 상황

`chromadb` 설치 중 다음 오류가 발생했다.

```text
Failed building wheel for chroma-hnswlib
error: Microsoft Visual C++ 14.0 or greater is required
```

## 원인

Windows + Python 3.12 환경에서 `chroma-hnswlib`의 사전 빌드 wheel을 사용하지 못하고 직접 컴파일하려고 했다.

하지만 Microsoft C++ Build Tools가 없거나, Python 버전 조합이 맞지 않아 빌드가 실패했다.

## 해결 방법

Python 3.11로 가상환경을 새로 생성한다.

```powershell
& "C:\Users\yoehe\AppData\Local\Programs\Python\Python311\python.exe" -m venv C:\AI_study\debugmate-rag\.venv
```

가상환경 활성화:

```powershell
(Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1)
```

Python 버전 확인:

```powershell
python --version
```

정상 예시:

```text
Python 3.11.9
```

그다음 설치한다.

```powershell
python -m pip install -r requirements.txt
```

## 주의사항

Chroma 설치가 필요하면 Python 3.11을 사용하는 것이 Windows에서 더 안정적이다.

`python --version`이 `Python 3.12`로 나오면 잘못된 가상환경을 사용 중일 수 있다.

## 관련 태그

chroma_error, chromadb, chroma_hnswlib, python311, python312, build_tools

# 오류 10: 가상환경 이름은 맞지만 Python 버전이 다름

## 상황

PowerShell 앞에는 `(.venv-1)`이 표시되지만, 실제 Python 버전은 3.12로 나왔다.

```powershell
python --version
```

결과:

```text
Python 3.12.10
```

## 원인

가상환경 이름은 `.venv-1`이지만, 해당 가상환경이 Python 3.12로 생성되었다.

가상환경 이름은 중요하지 않고, 내부 Python 버전이 중요하다.

## 해결 방법

현재 Python 실행 파일을 확인한다.

```powershell
python -c "import sys; print(sys.executable)"
```

Python 3.11 실행 파일로 가상환경을 새로 만든다.

```powershell
& "C:\Users\yoehe\AppData\Local\Programs\Python\Python311\python.exe" -m venv C:\AI_study\debugmate-rag\.venv
```

새 가상환경 활성화 후 다시 확인한다.

```powershell
python --version
python -c "import chromadb; import langchain_chroma; print('Chroma 설치 성공')"
```

## 주의사항

가상환경 이름보다 중요한 것은 다음 두 가지이다.

```text
python --version
python -c "import sys; print(sys.executable)"
```

## 관련 태그

python_env, venv, python311, python312, chroma
