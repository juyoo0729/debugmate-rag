# DebugMate RAG Jupyter / Python 환경 오류 기록 데이터셋

이 문서는 Jupyter Notebook, VS Code, Python 가상환경, pip 설치 경로, 커널 선택 문제를 정리한 RAG 검색용 문서이다.

# 오류 1: Jupyter Notebook에서 설치한 패키지를 못 찾음

## 상황

PowerShell에서 패키지를 설치했는데 Jupyter Notebook에서는 다음 오류가 발생한다.

```text
ModuleNotFoundError: No module named 'datasets'
```

또는:

```text
ModuleNotFoundError: No module named 'ragas'
```

## 원인

PowerShell에서 사용하는 Python과 Jupyter Notebook 커널이 사용하는 Python이 다를 수 있다.

예를 들어 PowerShell은 프로젝트 `.venv`를 사용하지만, Notebook은 전역 Python을 사용할 수 있다.

## 확인 방법

Jupyter Notebook 셀에서 현재 Python 경로를 확인한다.

```python
import sys
print(sys.executable)
```

PowerShell에서도 Python 경로를 확인한다.

```powershell
python -c "import sys; print(sys.executable)"
```

두 경로가 다르면 서로 다른 Python 환경을 사용 중이다.

## 해결 방법

프로젝트 가상환경에 Jupyter 커널을 등록한다.

```powershell
python -m pip install ipykernel
python -m ipykernel install --user --name debugmate-rag --display-name "Python (debugmate-rag)"
```

그다음 VS Code 또는 Jupyter Notebook에서 커널을 `Python (debugmate-rag)`로 선택한다.

## 주의사항

패키지 설치 후에는 Notebook 커널을 재시작해야 한다.

## 관련 태그

jupyter, kernel, python_env, package_error, ipykernel

# 오류 2: VS Code에서 잘못된 Python Interpreter를 사용함

## 상황

PowerShell에서는 패키지가 설치되어 있는데 VS Code Notebook이나 Python 파일 실행 시 패키지를 찾지 못한다.

## 원인

VS Code가 프로젝트 가상환경이 아니라 다른 Python interpreter를 선택하고 있다.

## 해결 방법

VS Code에서 다음 순서로 interpreter를 변경한다.

```text
Ctrl + Shift + P
→ Python: Select Interpreter
→ C:\AI_study\debugmate-rag\.venv\Scripts\python.exe 선택
```

또는 실제 성공한 가상환경이 `.venv-1`이라면 다음을 선택한다.

```text
C:\AI_study\debugmate-rag\.venv-1\Scripts\python.exe
```

## 확인 방법

VS Code 터미널 또는 Notebook에서 다음을 실행한다.

```python
import sys
print(sys.executable)
```

정상 예시:

```text
C:\AI_study\debugmate-rag\.venv\Scripts\python.exe
```

## 주의사항

가상환경 이름보다 중요한 것은 실제 Python 경로와 버전이다.

`python --version`과 `sys.executable`을 확인해야 한다.

## 관련 태그

vscode, interpreter, python_env, virtualenv

# 오류 3: python -m venv 명령을 Notebook 셀에서 실행함

## 상황

Jupyter Notebook 코드 셀에서 다음 명령어를 실행했다.

```python
python -m venv .venv
```

오류 메시지:

```text
SyntaxError: invalid syntax
```

## 원인

`python -m venv .venv`는 Python 코드가 아니라 터미널 명령어이다.

Jupyter Notebook 코드 셀에 그대로 입력하면 Python 문법으로 해석되어 오류가 발생한다.

## 해결 방법

PowerShell에서 실행한다.

```powershell
cd C:\AI_study\debugmate-rag
python -m venv .venv
```

Notebook 셀에서 터미널 명령어를 실행하려면 앞에 `!`를 붙일 수 있다.

```python
!python -m venv .venv
```

하지만 가상환경 생성은 PowerShell에서 하는 것이 더 안전하다.

## 주의사항

Python 코드와 터미널 명령어를 구분해야 한다.

Python 코드 예시:

```python
import pandas as pd
```

PowerShell 명령어 예시:

```powershell
python -m pip install pandas
```

## 관련 태그

syntax_error, jupyter, powershell, venv

# 오류 4: PowerShell에서 Activate.ps1을 찾지 못함

## 상황

가상환경을 활성화하려고 다음 명령어를 실행했다.

```powershell
.\.venv\Scripts\Activate.ps1
```

오류 메시지:

```text
The term '.\.venv\Scripts\Activate.ps1' is not recognized as a name of a cmdlet, function, script file, or executable program.
```

## 원인

다음 중 하나이다.

1. `.venv` 폴더가 없다.
2. 가상환경 생성이 실패했다.
3. 현재 PowerShell 위치가 프로젝트 폴더가 아니다.
4. `.venv` 폴더를 VS Code에서 수동으로 만들었다.
5. 실제 가상환경 이름이 `.venv-1`이다.

## 확인 방법

현재 위치를 확인한다.

```powershell
pwd
```

활성화 파일 존재 여부를 확인한다.

```powershell
Test-Path C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1
```

정상 결과:

```text
True
```

## 해결 방법

가상환경을 명령어로 다시 만든다.

```powershell
python -m venv C:\AI_study\debugmate-rag\.venv
```

활성화한다.

```powershell
(Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1)
```

## 주의사항

`.venv` 폴더를 직접 새 폴더로 만들면 안 된다.

반드시 `python -m venv` 명령어로 생성해야 한다.

## 관련 태그

powershell, activate, venv, python_env

# 오류 5: PowerShell 실행 정책 때문에 가상환경 활성화가 안 됨

## 상황

가상환경을 활성화하려고 할 때 다음 오류가 발생한다.

```text
running scripts is disabled on this system
```

## 원인

Windows PowerShell의 실행 정책이 `.ps1` 스크립트 실행을 제한하고 있다.

## 해결 방법

현재 PowerShell 세션에서만 실행 정책을 완화한다.

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```

그다음 가상환경을 활성화한다.

```powershell
.\.venv\Scripts\Activate.ps1
```

한 줄로 실행할 수도 있다.

```powershell
(Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1)
```

## 주의사항

`-Scope Process`는 현재 PowerShell 창에서만 적용된다.

보안 정책을 시스템 전체에 영구 변경하지 않아도 된다.

## 관련 태그

powershell, execution_policy, activate, venv

# 오류 6: 가상환경 이름은 맞지만 Python 버전이 다름

## 상황

터미널 앞에는 `(.venv-1)`이 표시되지만 Python 버전을 확인하면 3.12가 나온다.

```powershell
python --version
```

결과:

```text
Python 3.12.10
```

## 원인

`.venv-1`이라는 이름의 가상환경이 Python 3.12로 만들어졌기 때문이다.

가상환경 이름은 중요하지 않고, 내부 Python 버전이 중요하다.

## 해결 방법

Python 3.11 실행 파일로 가상환경을 새로 만든다.

```powershell
& "C:\Users\yoehe\AppData\Local\Programs\Python\Python311\python.exe" -m venv C:\AI_study\debugmate-rag\.venv
```

활성화 후 확인한다.

```powershell
python --version
python -c "import sys; print(sys.executable)"
```

정상 예시:

```text
Python 3.11.9
C:\AI_study\debugmate-rag\.venv\Scripts\python.exe
```

## 주의사항

Chroma를 사용할 경우 Windows에서는 Python 3.11 환경이 더 안정적이었다.

`python --version` 확인 없이 패키지를 설치하면 3.12 환경에 잘못 설치할 수 있다.

## 관련 태그

python311, python312, venv, chroma, python_env

# 오류 7: py -3.11 명령어가 동작하지 않음

## 상황

PowerShell에서 다음 명령어를 실행했지만 Python 3.11을 찾지 못한다.

```powershell
py -3.11 -m venv .venv
```

오류 메시지:

```text
No suitable Python runtime found
```

## 원인

Python 3.11이 설치되어 있지 않거나, Python Launcher가 Python 3.11을 인식하지 못하는 상태이다.

## 확인 방법

설치된 Python 목록을 확인한다.

```powershell
py -0p
```

정상 예시:

```text
-V:3.11 C:\Users\yoehe\AppData\Local\Programs\Python\Python311\python.exe
-V:3.12 C:\Users\yoehe\AppData\Local\Programs\Python\Python312\python.exe
```

## 해결 방법

Python 3.11이 설치되어 있다면 절대경로로 실행한다.

```powershell
& "C:\Users\yoehe\AppData\Local\Programs\Python\Python311\python.exe" -m venv C:\AI_study\debugmate-rag\.venv
```

Python 3.11이 없다면 설치한다.

```powershell
winget install -e --id Python.Python.3.11
```

설치 후 PowerShell을 다시 열고 확인한다.

```powershell
py -0p
```

## 관련 태그

python_launcher, python311, venv, powershell

# 오류 8: pip install 출력이 너무 많음

## 상황

`python -m pip install -r requirements.txt`를 실행했더니 출력이 너무 길어서 오류를 찾기 어렵다.

## 원인

pip는 의존성 패키지를 설치하면서 많은 로그를 출력한다.

정상 설치 중에도 수백 줄이 출력될 수 있다.

## 해결 방법

설치 로그를 파일로 저장한다.

```powershell
python -m pip install -r requirements.txt *> install.log
```

설치가 실패하면 마지막 40줄만 확인한다.

```powershell
Get-Content install.log -Tail 40
```

## 주의사항

pip 설치 오류의 핵심 원인은 보통 마지막 20~40줄에 있다.

전체 로그를 다 읽을 필요는 없다.

## 관련 태그

pip, install_log, powershell, debugging

# 오류 9: pip install은 성공했지만 import가 안 됨

## 상황

패키지를 설치했다고 생각했는데 Python에서 import하면 오류가 발생한다.

```text
ModuleNotFoundError: No module named 'chromadb'
```

## 원인

다음 중 하나일 수 있다.

1. 다른 Python 환경에 설치했다.
2. 설치 중간에 실패했지만 성공으로 착각했다.
3. 가상환경이 활성화되지 않은 상태에서 import했다.
4. VS Code가 다른 interpreter를 사용한다.

## 확인 방법

현재 Python 실행 경로를 확인한다.

```powershell
python -c "import sys; print(sys.executable)"
```

설치된 패키지를 확인한다.

```powershell
python -m pip show chromadb
```

또는 전체 목록을 확인한다.

```powershell
python -m pip list
```

## 해결 방법

올바른 가상환경을 활성화한 뒤 설치한다.

```powershell
(Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned) ; (& C:\AI_study\debugmate-rag\.venv\Scripts\Activate.ps1)
python -m pip install -r requirements.txt
```

설치 확인:

```powershell
python -c "import chromadb; import langchain_chroma; print('Chroma 설치 성공')"
```

## 관련 태그

pip, import_error, module_not_found, python_env

# 오류 10: 가상환경에서 일반 PowerShell로 나오고 싶음

## 상황

PowerShell 앞에 다음처럼 표시되어 있다.

```text
(.venv) PS C:\AI_study\debugmate-rag>
```

또는:

```text
(.venv-1) PS C:\AI_study\debugmate-rag>
```

사용자는 일반 PowerShell 상태로 돌아가고 싶다.

## 해결 방법

다음 명령어를 실행한다.

```powershell
deactivate
```

정상적으로 빠져나오면 앞의 `(.venv)` 표시가 사라진다.

```text
PS C:\AI_study\debugmate-rag>
```

## 주의사항

가상환경은 프로젝트 작업용 Python 환경이다.

패키지 설치, Streamlit 실행, RAG 코드 실행은 가상환경 안에서 하는 것이 좋다.

Git 명령어는 일반 PowerShell에서도 실행 가능하다.

## 관련 태그

venv, deactivate, powershell, python_env
